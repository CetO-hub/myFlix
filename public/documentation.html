Documentation

The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Business logic Endpoint URL Method Request parameters Request format Response format
Return a list of ALL movies to the user http//:localhost:8080/movies GET none none A JSON object with the respective data for all movies(_id: ObjectId, Title: String, Description: String, Genre: {Name: String, Description: String}, Director: {Name: String, Bio: String, Birth: String, Death: String}, ImagePath: String, Featured: Boolean).
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user http//:localhost:8080/movies/:title GET :title none A JSON object with the respective data for a single movie(_id: ObjectId, Title: String, Description: String, Genre: {Name: String, Description: String}, Director: {Name: String, Bio: String, Birth: String, Death: String}, ImagePath: String, Featured: Boolean).
Return data about a genre (description) by name/title (e.g., “Thriller”) http//:localhost:8080/movies/:title/genre GET :title none A JSON object holding data about the genre of a single movie (Genre: {Name: String, Description: String}, _id: ObjectId)
Return data about a director (Name, Bio, Birth, Death) by name http//:localhost:8080/directors/:name GET :name none A JSON object holding all data about a director (Director: {Name: String, Bio: String, Birth: String, Death: String}, _id: ObjectId)
Allow new users to register http//:localhost:8080/users POST none A JSON object with the respective data to be added (Username: String(required), Password: String(required), Email: String(required), Birthday: Date(optional)}) A JSON object with the respective data (_id: ObjectId, Username: String, Password: String, Email: String, Birthday: Date(optional), FavoriteMovies: [ObjectId(ref: Movie)])
Allow users to update their user info (username, password, email and birthday) http//:localhost:8080/users/:id PUT :id A JSON object with the respective data to be updated (Username: String(required), Password: String(required), Email: String(required), Birthday: Date(optional)}) A JSON object with the respective data (_id: ObjectId, Username: String, Password: String, Email: String, Birthday: Date(optional), FavoriteMovies: [ObjectId(ref: Movie)])
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) http//:localhost:8080/users/:MovieID POST :MovieID A JSON object holding the data for the username (Username: String}) A JSON object with the respective data (_id: ObjectId, Username: String, Password: String, Email: String, Birthday: Date(optional), FavoriteMovies: [ObjectId(ref: Movie)])
Allow users to remove a movie from their list of favorites http//:localhost:8080/users/:MovieID/:username DELETE :MovieID, :username None A JSON object with the respective data (_id: ObjectId, Username: String, Password: String, Email: String, Birthday: Date(optional), FavoriteMovies: [ObjectId(ref: Movie)])
Allow existing users to deregister http//:localhost:8080/users/:username DELETE :username none A text message that the user has been deleted
